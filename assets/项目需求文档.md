# 热点电商导购系统 - Web 应用

## 项目目标
搭建一个公网可访问的 Web 应用，展示"热点 → 消费场景 → 爆款商品"的 AI 导购系统。用户能看到：左侧是简化版京东商品页（像素风），右侧是后台 4 Agent 实时工作日志。

## 整体架构

**4 Agent 流水线**（周期性自动运行 + 用户可手动触发）：
1. **感知 Agent** — 抓取网络热点 → LLM 提取消费场景 → 入场景库
2. **挂品 Agent** — 根据场景在商品库匹配商品 → 建立场景-商品关联
3. **导购生成 Agent** — 为挂品结果生成推荐理由 + 推荐标签 → 入推荐库
4. **分发 Agent** — 响应用户首次进入/搜索请求 → 返回推荐结果

**流水线触发**：感知 → 挂品 → 导购（单向串联），完成后入分发表。分发是查询型（响应前端）。

### 整体布局
- **左侧 70%**：商品推荐区（简化版京东）
- **右侧 30%**：4 Agent 实时工作日志（实时刷新）
- **顶部固定**：Logo + 搜索框
- **风格**：像素艺术、硬边角、复古配色（深色或浅色像素风），等宽字体

### 左侧商品区
- 商品卡片网格（每行 3-4 个）
- 每张卡片：
  - 像素风占位图（用 CSS 实现：纯色块 + 像素图案 + emoji 抽象，不用真实图片避免慢加载）

  - 价格（突出显示）
  - 店铺名
  - 推荐标签（"🔥 高分推荐"、"💎 性价比"、"⚡ 新品"等）
  - 推荐理由（hover 时显示）
- 鼠标 hover 卡片有像素风高亮效果

### 右侧 Agent 日志栏
- 4 个 Agent 卡片堆叠显示：
  - **感知 Agent** 🛰️
  - **挂品 Agent** 🛒
  - **导购生成 Agent** ✍️
  - **分发 Agent** 📡
- 每个 Agent 卡片显示：
  - Agent 名称 + 像素风图标
  - 状态指示器（运行中=🟢、空闲=⚪、完成=✅、失败=🔴）
  - 最近 5 条日志（实时滚动）
  - 当前任务描述

### 顶部导航
- Logo 文字（像素风："热点商城"或类似）
- 搜索框（占位文字："搜商品或场景关键词..."）
- 搜索按钮（像素风）

## 后端 API 设计

| 路径 | 方法 | 说明 |
|---|---|---|
| `/api/recommend` | GET | 首页推荐（首次进入/刷新调用） |
| `/api/search?q=xxx` | GET | 搜索推荐（搜商品名 + 场景关键词） |
| `/api/agents/logs` | GET | 4 Agent 最近日志（前端每 3 秒轮询） |
| `/api/agents/status` | GET | 4 Agent 当前状态（running/idle/done/failed） |
| `/api/run` | POST | 手动触发一次完整流水线 |
| `/api/run/status` | GET | 查询流水线当前执行状态 |

## 数据库表设计

### scenes（场景库）
- id (PK, auto)
- title (场景标题)
- description (场景描述)
- target_user (目标用户)
- confidence (置信度 0-1)
- keywords (JSON 数组，搜索关键词)
- source_hotspot (来源热点标题)
- created_at (创建时间)
- expires_at (时效过期时间，72 小时后)

### products（商品库）
- id (PK, auto)
- sku_id (商品 SKU)
- title (商品标题)
- price (价格)
- original_price (原价)
- shop_name (店铺名)
- good_rate (好评率)
- sales (销量)
- category (类目)
- icon_emoji (像素风图标 emoji)
- bg_color (背景色 hex)

### scene_products（场景-商品关联）
- id (PK, auto)
- scene_id (FK)
- product_id (FK)
- match_score (匹配分 0-1)
- created_at

### recommendations（推荐内容库）
- id (PK, auto)
- scene_id (FK)
- product_id (FK)
- reason (推荐理由，LLM 生成)
- tags (JSON 数组，如 ["高性价比", "新品", "爆款潜力"])
- score (综合评分)
- created_at

### agent_logs（Agent 日志表）
- id (PK, auto)
- agent_name (感知/挂品/导购/分发)
- level (info/warn/error)
- message (日志内容)
- created_at

### agent_status（Agent 状态表）
- agent_name (PK)
- status (running/idle/done/failed)
- current_task (当前任务描述)
- last_run_at (最后运行时间)

## 4 Agent 实现细节（演示模式）

### 感知 Agent
- mock 抓取：生成 5-8 个假热点（如"露营经济持续升温"、"防晒服夏季热销"等）
- mock LLM：每个热点生成 2-3 个消费场景（如"露营装备升级场景"、"户外防晒场景"）
- 入库：场景写入 scenes 表
- 判重：title 相同则不重复入库
- 时效：72 小时过期

### 挂品 Agent
- 读取最近 24 小时新增的 scenes
- mock 匹配：从 products 表中随机选 5-8 个商品建立关联
- 匹配分：基于类目相关性和商品质量（mock 时随机生成 0.6-0.95）
- 写入 scene_products 表

### 导购生成 Agent
- 读取 scene_products 中 score 最高的 5 个关联
- mock 生成推荐理由（从模板中选，如"针对{场景}，这款{商品}有{亮点}，是{人群}的优质选择"）
- 生成推荐标签（"高性价比"、"新品"、"爆款潜力"、"全网热销"等 2-3 个）
- 写入 recommendations 表

### 分发 Agent
- 不主动跑，是被动查询
- 响应 /api/recommend：从 recommendations 表取 score top 10
- 响应 /api/search：搜商品名（products.title LIKE）和场景关键词（scenes.keywords），结果合并去重

## 实时日志机制

每次流水线运行时：
1. Agent 开始 → agent_status.status = 'running' + 写 agent_logs
2. Agent 执行中 → 每完成一步写一条 agent_logs
3. Agent 结束 → agent_status.status = 'done' 或 'failed'

前端每 3 秒轮询 `/api/agents/logs` 和 `/api/agents/status`，把新日志追加到右侧栏。

## 调度机制

- 后台每 1 小时自动触发一次完整流水线（演示模式可缩短为每 5 分钟，方便看到效果）
- 用户可以点"立即跑一次"按钮手动触发
- 演示模式：项目启动后立即跑一次，之后每 5 分钟跑一次

## 技术要求

- **前端**：原生 HTML + CSS + JavaScript（不用框架，零构建步骤）
- **后端**：可以用 Node.js (Express/Koa) 或 Python (FastAPI)，根据 Coze 编程支持选择
- **数据库**：Supabase（如果 Coze 编程支持）或 SQLite
- **样式**：像素风格 CSS（硬边角 box-shadow、像素字体、像素图案）
- **图标**：用 emoji 代替矢量图
- **不要用**：Tailwind、React、Vue 等需要构建的框架
- **部署**：自动部署到公网，无需登录控制

## 注意事项

- 项目类型用 `web`
- 不需要登录控制
- 演示数据要够丰富，让用户第一次进入就能看到效果
- 像素风格要统一，整站视觉一致
- 跑流水线时实时日志要"演得真实"，比如"正在抓取百度热搜..."、"已抓取 50 条热点..."、"开始提取消费场景..."、"场景已入库: xxx"
- 跑完一次流水线后，前端要能立即看到新的商品推荐
